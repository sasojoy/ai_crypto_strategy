# H16 Strategy Development Protocol (ZTDP)

## 1. Reporting Statutory Format
All backtest reports must include:
- **Strategy Manifest Table**: Generated by `AuditManager`.
- **Physical Evidence**: `head -n 5` of `backtest_results.csv`.
- **Execution Logs**: Progress logs for complex backtests (e.g., WFO).
- **Friction Audit**: Mandatory 0.04% fee and 0.05% slippage.

## 2. Development Prohibitions
- **No Hallucinations**: All data must be backed by physical CSV content.
- **Data Isolation**: `backtest_results.csv` must be physically deleted before each run.
- **Unified Wallet**: All symbols must share a single `TradingAccount` instance in chronological order.

## 3. Audit Standards
- **Volume Z-Score**: Must use standard deviation-based calculation.
- **Logic Drift**: Any change in fees or slippage must trigger a RED WARNING.
